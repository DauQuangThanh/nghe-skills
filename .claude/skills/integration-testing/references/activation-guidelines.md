# Activation Guidelines


This skill should be activated when:
- Designing integration test strategies
- Writing API integration tests
- Testing database interactions
- Testing microservices communication
- Implementing E2E tests
- Setting up test automation
- Configuring CI/CD test pipelines
- Creating test data strategies
- Implementing contract testing
- Testing distributed systems
- Debugging integration issues
- Improving test coverage

The skill provides the most value when given system architecture diagrams, API specifications, and clear testing objectives.
